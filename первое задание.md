## ШАГ 7
Создаю EtherChannek между коммутаторами в новосибирске используя интерфейсы G0/1, G0/2 не нарушая требования

На rus-nsk-sw0 (активная сторона):
```
enable
configure terminal
interface range fastEthernet 0/5-6
channel-group 1 mode active
channel-protocol lacp
exit
interface port-channel 1
switchport mode trunk
```

На rus-nsk-sw1 (пассивная сторона):
```
enable
configure terminal
interface range fastEthernet 0/1-2
channel-group 1 mode passive
channel-protocol lacp
exit
interface port-channel 1
switchport mode trunk
```
При настройке LACP одна сторона должна выступать в роли активной , а другая — в роли пассивной. Согласно заданию, коммутатор SW0 является ответственным за инициирование согласования, поэтому он настроен как активный, а SW1 — как пассивный.

```

1024

transport input ssh

```
> Для этих действий понадобилось создать ключи RSA, команды для их создания
> crypto key generate rsa
> 1024

Для удаленного доступа только для ssh используются команды:
```
line vty 0 4
transport input ssh
```
line vty 0 4 — выбирает линии 0-4 (всего 5 одновременных удаленных подключений)
transport input ssh — разрешает только SSH, Telnet запрещен

## ШАГ 13
Настраиваю интерфейсы f0/2, f0/3, f0/4

На SW0:
```
enable
configure terminal
interface range fastEthernet 0/2-4
spanning-tree portfast
spanning-tree bpduguard enable
no cdp enable
switchport port-security
switchport port-security maximum 1
switchport port-security mac-address sticky
switchport port-security violation shutdown
exit
```
На SW1:
```
enable
configure terminal
interface range fastEthernet 0/3-5
spanning-tree portfast
spanning-tree bpduguard enable
no cdp enable
switchport port-security
switchport port-security maximum 1
switchport port-security mac-address sticky
switchport port-security violation shutdown
exit
```
команды отвечающие за требования:
- spanning-tree portfast - Переводит порт в режим PortFast. Порт сразу переходит в состояние пересылки.
- spanning-tree bpduguard enable - Включает BPDU Guard. Если порт получит BPDU-кадр, он автоматически перейдет в состояние err-disable.
- no cdp enable - Отключает CDP на порту.
- switchport port-security violation shutdown - Если на порт попытается подключиться другое устройство или превышено максимальное количество MAC-адресов, порт перейдет в состояние err-disable
